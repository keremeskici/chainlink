02 / 11 [ THE PROBLEM ]

UMA's Oracle Is
Riggable By Design.

Core Flaw — Conflict of Interest

  • UMA token holders act as final judges in market disputes
  • Whales can simultaneously hold large bets and control the vote outcome
  • If rig-profit > 0.05% slashing penalty → whale votes false
  • Most retail holders never vote — economically irrational

Side Problem — Gas Barrier

  • Gas spikes make voting uneconomic for small holders
  • Participation collapses further, amplifying whale dominance
  • Compounds bias: fewer voters = cheaper to rig

⚠ WHALE HOLDS BET → WHALE CONTROLS VOTE → MARKET RESOLVES FALSE → USERS LOSE FUNDS

07 / 11 [ CHAINLINK COMPONENTS ]

Chainlink.
Every Layer. Every Guarantee.

Confidential HTTP (CRE)
01 — Tamper-Proof AI Calls
Chainlink CRE executes HTTP requests to AI provider APIs inside a Trusted Execution Environment (TEE). The request payload, headers, and response are cryptographically attested — no man-in-the-middle attack is possible, period.
⬡ BENEFIT — AI responses are provably unmodified before hitting on-chain logic
Standard HTTP — News Fetching
02 — Real-World Fact Grounding
Before voting, the CRE fetches live articles from verified outlets — Reuters, CNN, BBC — via Chainlink Standard HTTP Jobs (news_fetcher.ts). HTML is stripped, truncated to 8k chars, and fed as context to every AI agent in the swarm.
⬡ BENEFIT — Agents vote on verifiable facts, not hallucinations
Chainlink Automation
03 — Trigger-Based Market Detection
Chainlink Automation monitors the UMA network for new market creation events. The moment a votable market appears, our pipeline is triggered automatically — no manual cron job, no centralised scheduler, no single point of failure.
⬡ BENEFIT — Zero missed votes; every market is caught as soon as it opens
OracleRegistry.sol
04 — Access-Controlled On-Chain Write
Only authenticated Chainlink nodes are whitelisted callers on OracleRegistry.sol. The consensus result written on-chain cannot be submitted by any other party — no front-running, no spoofing, no governance attack via fake oracle data.
⬡ BENEFIT — Smart contract receives only cryptographically verified consensus answers

09 / 11 [ SOURCE WEIGHT ALGORITHM ]

Not All Sources
Are Equal.

How Weights Work

Every news source in the pool carries a credibility weight derived from its historical accuracy on similar market types. The AI swarm's consensus vote is weighted — not just counted — by source quality.

Weight by Market Type

  • Political & geopolitical — Reuters, AP, BBC, CNN, local gov feeds, Telegram channels
  • Weather — AccuWeather, Weather24, NOAA, Weather Underground
  • Financial — Bloomberg, FT, WSJ, on-chain price feeds
  • Sports — Official league APIs, ESPN, BBC Sport
HIGH WEIGHT
  • Reuters
  • BBC / CNN
  • AP News
  • AccuWeather
  • Official gov feeds
LOW WEIGHT
  • Reddit
  • 4chan / anonymous boards
  • Unverified Telegram
  • Personal blogs
  • Social media posts

⚠ LOW-WEIGHT SOURCE INCLUDED BUT CANNOT SWING CONSENSUS — IT CAN ONLY CORROBORATE A HIGH-WEIGHT SIGNAL

Why This Matters

Reddit and 4chan often spread unconfirmed rumours hours before established outlets. Including them at full weight would bias the vote. Including them at minimal weight retains early signal while preventing manipulation.
